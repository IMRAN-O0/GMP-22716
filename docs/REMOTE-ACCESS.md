# 🔐 الوصول البعيد الآمن (Tailscale / WireGuard)

نظام GMP مصمَّم للعمل داخل شبكة LAN عبر HTTP. لتمكين الوصول من **خارج المصنع** (مدير في بيته، فرع آخر) **دون تعريض النظام للإنترنت ودون نقل البيانات بشكل غير مشفّر**، نستخدم شبكة خاصة افتراضية مشفّرة (VPN).

> **الفكرة الأساسية:** بدلاً من فتح منفذ للعالم، نُنشئ نفقاً مشفّراً يجعل الجهاز البعيد وكأنه داخل شبكة المصنع. المهاجم على الإنترنت **لا يرى النظام أصلاً** — وهذا أقوى من مجرد إضافة HTTPS.

---

## ⚙️ تهيئة دعم الـ VPN في النظام (تمت برمجياً)

عناوين Tailscale/WireGuard تقع في نطاق `100.64.0.0/10`. تم **تحديث فحص CORS** في `server.ts` و `server-electron.ts` ليقبل هذا النطاق تلقائياً، فلا حاجة لأي تعديل إضافي في الكود.

اختيارياً، يمكنك تثبيت العنوان صراحةً في `.env`:
```env
# عنوان جهاز السيرفر على شبكة Tailscale (يبدأ بـ 100.x.x.x)
ALLOWED_ORIGINS=http://100.101.102.103:3009,http://localhost:3009
```

---

## 🟢 الخيار أ: Tailscale (الأسهل — موصى به)

Tailscale طبقة سهلة فوق WireGuard، مجانية حتى 100 جهاز، ولا تحتاج IP ثابتاً أو إعداد جدار حماية.

### 1) على جهاز السيرفر (حيث يعمل النظام)
1. ثبّت Tailscale من <https://tailscale.com/download>
2. سجّل الدخول: `tailscale up`
3. اعرف عنوان الجهاز على الشبكة:
   ```bash
   tailscale ip -4        # مثال: 100.101.102.103
   ```
4. شغّل النظام كالمعتاد:
   ```bash
   npm run build:all && npm start
   ```

### 2) على الأجهزة البعيدة (مدير / فرع آخر)
1. ثبّت Tailscale وسجّل الدخول بنفس الحساب/الشبكة (Tailnet).
2. افتح المتصفح على عنوان السيرفر:
   ```
   http://100.101.102.103:3009
   ```
   الاتصال الآن **مشفّر بالكامل** عبر نفق WireGuard، رغم أنه HTTP داخل النفق.

### 3) (اختياري) اسم بدل رقم — MagicDNS
فعّل **MagicDNS** من لوحة تحكم Tailscale، فتصل عبر اسم بدل الرقم:
```
http://gmp-server:3009
```

### 4) (اختياري) HTTPS حقيقي مجاناً عبر Tailscale Serve
يوفّر Tailscale شهادة HTTPS تلقائية لأجهزة شبكتك:
```bash
tailscale serve --bg 3009
# يصبح الوصول: https://gmp-server.<your-tailnet>.ts.net
```
بهذا تحصل على **VPN + HTTPS معاً** دون أي تعديل في الكود.

## 🔵 الخيار ب: WireGuard يدوي (تحكّم كامل، بلا طرف ثالث)

مناسب إن أردت استضافة الـ VPN بنفسك دون الاعتماد على خدمة Tailscale.

### 1) على السيرفر
```bash
# Ubuntu/Debian
sudo apt install wireguard
wg genkey | tee server_private.key | wg pubkey > server_public.key
```
أنشئ `/etc/wireguard/wg0.conf`:
```ini
[Interface]
Address = 10.8.0.1/24
ListenPort = 51820
PrivateKey = <محتوى server_private.key>

[Peer]                       # كل جهاز بعيد = Peer
PublicKey = <المفتاح العام للجهاز البعيد>
AllowedIPs = 10.8.0.2/32
```
```bash
sudo wg-quick up wg0
sudo ufw allow 51820/udp     # المنفذ الوحيد المفتوح للعالم (UDP فقط)
```

### 2) على الجهاز البعيد
```ini
[Interface]
Address = 10.8.0.2/24
PrivateKey = <المفتاح الخاص للجهاز البعيد>

[Peer]
PublicKey = <محتوى server_public.key>
Endpoint = <IP-العام-للسيرفر>:51820
AllowedIPs = 10.8.0.0/24
PersistentKeepalive = 25
```
ثم صِل وافتح: `http://10.8.0.1:3009`

> ملاحظة: عنوان `10.8.0.x` يقع ضمن نطاق `10.x` المسموح به أصلاً في فحص CORS.

## 🛡️ ملاحظات أمنية (مهمة لتدقيق GMP)

1. **لا تفتح المنفذ 3009 للإنترنت إطلاقاً.** المنفذ الوحيد المكشوف يجب أن يكون منفذ WireGuard (`51820/udp`) — أو لا شيء مع Tailscale.
2. **غيّر كلمة مرور admin الافتراضية** و `JWT_SECRET` فور التشغيل (انظر README).
3. **مبدأ أقل صلاحية:** أضف فقط الأجهزة الموثوقة إلى شبكة Tailnet / قائمة Peers.
4. **للتشفير داخل الـ LAN نفسها** (لا عبر VPN فقط): يمكن لاحقاً إضافة HTTPS عبر `Tailscale Serve` أو Caddy أو HTTPS مدمج في Express.
5. **سجّل الوصول البعيد** ضمن إجراءات الجودة لديك (من يصل، متى) — مفيد للتدقيق الرقابي.

## ✅ الخلاصة

| الطريقة | التشفير | منفذ مكشوف للعالم | HTTPS فعلي |
|---|---|---|---|
| Tailscale | ✅ WireGuard | ❌ لا شيء | اختياري (Serve) |
| WireGuard يدوي | ✅ WireGuard | منفذ UDP واحد | لا |

**التوصية:** ابدأ بـ **Tailscale + `tailscale serve`** للحصول على VPN مشفّر و HTTPS معاً، بأقل جهد وأقل سطح هجوم.
