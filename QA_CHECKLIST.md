# NAXE OS — قائمة فحص ما قبل الإطلاق (Pre-Launch QA)

> الطبقة ١ (اختبارات API الآلية) تُشغّل بـ `npm test`. هذا الملف يغطّي الطبقة ٢
> (فحص الواجهة يدوياً) والطبقة ٣ (تثبيت ما قبل الإطلاق). ضع ✅ أمام كل بند بعد التحقق.

---

## ٠) تشغيل الفحوصات الآلية (يجب أن تكون كلها خضراء)
- [ ] `npm run typecheck` — لا أخطاء TypeScript.
- [ ] `npm run lint` — لا أخطاء.
- [ ] `npm test` — كل الاختبارات تنجح (وحدات + تكامل API).
- [ ] `npm run build` و `npm run build:server` — البناء ينجح.

---

## ١) المصادقة والصلاحيات
- [ ] تسجيل الدخول بحساب admin يعمل، وكلمة مرور خاطئة تُرفض.
- [ ] **تغيير كلمة مرور admin الافتراضية** قبل التسليم للعميل.
- [ ] مستخدم مستوى ٤ لا يرى زر «حفظ واعتماد» (يرى «إرسال للمراجعة») فقط.
- [ ] مستخدم مقيّد بقسم لا يرى أقسام غيره في القائمة الجانبية (403 عند محاولة الدخول للرابط مباشرة).
- [ ] تسجيل الخروج يمسح الجلسة ويعيد لصفحة الدخول.

## ٢) فحص النماذج لكل قسم (لكل نموذج: تعبئة → مسودة → اعتماد → عرض → طباعة)
لكل قسم افتح نموذجاً واحداً على الأقل ونفّذ الدورة الكاملة:
- [ ] **الموارد البشرية والتدريب (HRT)** — ملف الموظف، طلب إجازة، تقييم أداء، إخلاء طرف.
- [ ] **المخزون (INV)** — طلب شراء، فاتورة استلام، حركة مخزون.
- [ ] **الإنتاج (PRD)** — أمر إنتاج، سجل التصنيع.
- [ ] **الجودة (QM)** — عدم مطابقة، CAPA، سجل بيئي.
- [ ] **المختبر (LAB)** — استلام عينة، نتائج اختبار.
- [ ] **التعبئة والتغليف (PKG)** — تصفية خط، فحص أثناء التعبئة.

لكل نموذج تحقّق من:
- [ ] الحقول الإلزامية (*) تمنع الإرسال عند تركها فارغة وتظهر رسالة عربية واضحة.
- [ ] الترقيم التلقائي (رقم السجل/الموظف) يتولّد ولا يتكرر.
- [ ] «حفظ كمسودة» ثم العودة للتعديل يحفظ القيم.
- [ ] العرض (FormViewer) يظهر كل الحقول بمسمياتها العربية الصحيحة (لا مفاتيح إنجليزية).
- [ ] الطباعة تُظهر الترويسة (شعار + اسم الشركة + رقم الترخيص) ورقم الصفحة في PDF.

## ٣) ميزات الموارد البشرية الجديدة
- [ ] حقلا «تاريخ انتهاء الإقامة» و«تاريخ انتهاء العقد» يُحفظان ويظهران في العرض.
- [ ] تنزيل نموذج Excel للموظفين يعمل ويفتح بصفّي العناوين + التعليمات.
- [ ] رفع ملف Excel معبّأ → معاينة صحيحة → استيراد → ظهور الموظفين في القائمة بأرقام تلقائية.
- [ ] رفع ملف بصف ناقص يُظهر خطأ الصف بوضوح.

## ٤) الترابط بين الأقسام (Workflow)
- [ ] اعتماد «ملف موظف» جديد يُنشئ تنبيهاً في القسم لإنشاء خطة تدريب.
- [ ] التقارير تُظهر البيانات المُدخلة فعلاً (أرصدة، شحنات، دفعات).
- [ ] تتبّع الدفعة (Batch Traceability) يربط الإنتاج بالمخزون والجودة.

## ٥) الواجهة والتوافق
- [ ] الاتجاه RTL سليم في كل الشاشات، ولا تتداخل العناصر.
- [ ] الشعار الجديد NAXE OS يظهر في الشريط الجانبي وشاشة الدخول وأيقونة التطبيق.
- [ ] التجربة سليمة على دقّات الشاشة الشائعة (1366×768 و1920×1080).

## ٦) تثبيت ما قبل الإطلاق (أمان وبنية) 🔒
- [x] **`JWT_SECRET`** ✅ مُنفّذ: الإنتاج يرفض الإقلاع إن كان المفتاح غائباً أو يساوي الافتراضي (Electron يولّده تلقائياً). للخادم: عيّن `JWT_SECRET` قوياً.
- [x] **كلمة مرور admin الافتراضية** ✅ مُنفّذ: يُجبر admin على تغييرها عند أول دخول عبر شاشة إلزامية.
- [x] **تفويض النماذج الحسّاسة على الخادم** ✅ مُنفّذ: العرض الوظيفي (F-HRT-007) وإخلاء الطرف (F-HRT-013) يتطلبان صلاحية صريحة لإنشائها/عرضها/تعديلها (عدا admin).
- [ ] **HTTPS**: عند النشر كخادم شبكة، تفعيل شهادة TLS (إعداد نشر — خارج الكود).
- [ ] **النسخ الاحتياطي**: التأكد من عمل النسخ التلقائي لقاعدة البيانات واختبار الاستعادة.
- [ ] اختبار ملف التثبيت `.exe` على جهاز Windows نظيف (تثبيت → تشغيل → إنشاء سجل → طباعة).

### كيفية تشغيل اختبارات API الآلية
```bash
npm test                 # كل الاختبارات (وحدات + تكامل)
npx vitest run tests/api.integration.test.ts   # اختبارات الـ API فقط
```
تشغّل الاختبارات الخادم الحقيقي على قاعدة بيانات مؤقتة وتفحص: المصادقة،
التحقق من الحقول، الاعتماد ومزامنة جدول الموظفين، الصلاحيات، الأمان،
والاستيراد الجماعي — عبر كل الأقسام.
